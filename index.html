SFTP-to-EAM Meter Reading Integration (NiFi + REST API)

I designed and implemented an end-to-end integration pipeline in Apache NiFi to automate the ingestion of asset meter readings into HxGN EAM. The flow retrieves raw data files from a secure SFTP location, validates and normalizes the content using Groovy scripting and JSONPath extraction, and converts multi-record files into individual meter-reading events. Each event is enriched with required metadata and transformed into a clean, structured JSON payload that aligns with HxGN EAM’s REST API specifications. The system then posts these payloads directly to the EAM API to create corresponding meter-reading records. The solution includes automated file archiving, detailed logging, and error routing for full traceability. This integration eliminated manual data entry, improved data consistency, and demonstrated my ability to build scalable, production-grade data flows using NiFi and RESTful services.

UDS Data Importer Tool

The User-Defined Screen Excel Import Tool is a custom utility in HxGN EAM that allows users to upload data into a UDS screen in bulk using a predefined Excel file. The tool reads each row from the Excel sheet, validates the values, maps them to the corresponding UDS fields, and inserts or updates the data in the UDS tables. For every row processed, it returns a success or error message, making it easy for users to identify and correct issues. This helps eliminate manual entry, speeds up data loading, and ensures accuracy during configuration or mass updates.
